Parte B · Bloco 4
O contrato inteiro · counter

Um contrato é estado + função

pragma language_version >= 0.23; import CompactStandardLibrary; // public state export ledger round: Counter; // transition function changing public state export circuit increment(): [] { round.increment(1); }

Sem witness, sem privacidade, ainda. É a base limpa, antes de adicionar o segredo.

Parte B · Bloco 4

A anatomia, linha por linha

cada linha é o modelo mental com palavras-chave.

pragma language_version >= 0.23; import CompactStandardLibrary; export ledger round: Counter; // estado público export circuit increment(): [] { // vira um circuito ZK round.increment(1); // muda o estado público }
1ledger = o estado público: on-chain, todo mundo lê. Counter é um tipo da stdlib (tem .increment()), não um número solto.
2circuit, não function. É o "roda o seu código" do seu lado: a chain só verifica a prova, não executa. O [] = não devolve nada; existe pelo efeito no estado.
3round.increment(1) muda o estado público, é isso que a chain registra.

Parte B · Bloco 5

O contrato inteiro

pragma language_version >= 0.23; import CompactStandardLibrary; // 1 · estado público export ledger note: Opaque<"string">; export ledger hasNote: Boolean; export ledger authorCommitment: Bytes<32>; // 2 · o segredo (a chave) witness local_secret_key(): Bytes<32>; // 3 · faz o carimbo pure circuit compute_author_commitment(sk: Bytes<32>): Bytes<32> { return persistentHash<Vector<2, Bytes<32>>>([pad(32, "bboard:author:"), sk]); } // 4 · postar export circuit postNote(content: Opaque<"string">): [] { assert(disclose(!hasNote), "Board is full"); const sk = local_secret_key(); const commitment = compute_author_commitment(sk); note = disclose(content); authorCommitment = disclose(commitment); hasNote = true; } // 5 · tirar export circuit takeDown(): [] { assert(disclose(hasNote), "No note to take down"); const sk = local_secret_key(); const callerCommitment = compute_author_commitment(sk); assert(disclose(callerCommitment == authorCommitment), "Only the original author"); hasNote = false; }
as 5 partes
1
estado público
a nota, se tem nota, e o autor virado um hash
2
o segredo
a chave do autor, que nunca sai da sua máquina
3
faz o carimbo
o hash da chave, que esconde quem postou
4
postar
guarda a nota e o carimbo, se o quadro está vazio
5
tirar
só quem refaz o carimbo com a mesma chave

O estado público

// estado público: todo mundo lê export ledger note: Opaque<"string">; // o conteúdo da nota export ledger hasNote: Boolean; // se já tem nota no quadro export ledger authorCommitment: Bytes<32>; // o autor, escondido num hash

A nota é pública. O autor, não: vira um hash. E de onde vem esse hash?

Parte B · Bloco 5

witness: a porta do dado privado

// witness: o dado privado que você fornece witness local_secret_key(): Bytes<32>; // calcula o commitment (hash) da chave do autor pure circuit compute_author_commitment(sk: Bytes<32>): Bytes<32> { return persistentHash<Vector<2, Bytes<32>>>([pad(32, "bboard:author:"), sk]); }

A chave nunca sai da sua máquina. O hash dela esconde quem é, mas só a mesma chave refaz ele.

Parte B · Bloco 5

Postar: conteúdo público, autor escondido

export circuit postNote(content: Opaque<"string">): [] { assert(disclose(!hasNote), "Board is full - someone already posted a note"); const sk = local_secret_key(); const commitment = compute_author_commitment(sk); note = disclose(content); authorCommitment = disclose(commitment); hasNote = true; }
1lê a chave privada (witness)
2o commitment vem da chave, então também é privado
3escrever no ledger torna público, por isso o disclose
4isso já era público, não precisa de disclose
Parte B · Bloco 5

disclose: você autoriza o que vira público

privado
contenta nota que você digita
sksua chave
commitmento hash da chave (interno)
→
disclose() você autoriza ✓
→
público
o compilador exige disclose quando o dado privado vira observável:
escrito no estado público (note, authorCommitment)
retornado por um circuit exportado
usado na condição de um assert
note = disclose(content); // a nota vira pública authorCommitment = disclose(commitment); // o carimbo, também

O compilador só rastreia de onde o dado veio, não a criptografia: ele cuida do caminho, e você responde pelo que é seguro revelar.

Parte B · Bloco 5

Provar autoria sem revelar a identidade

export circuit takeDown(): [] { assert(disclose(hasNote), "No note to take down"); const sk = local_secret_key(); const callerCommitment = compute_author_commitment(sk); assert(disclose(callerCommitment == authorCommitment), "Only the original author can take down the note"); hasNote = false; }
authorCommitment
guardado no quadro
==
callerCommitment
refeito com a sua chave
→
✓bateé o autor

A chave nunca sai da sua máquina: pra chain cruza só o sim/não. É o padrão commitment, publica o hash uma vez e prova depois recomputando.

Parte B · Bloco 5
framework de teste · simulador

Testar a lógica sem deploy

it("preserva a privacidade do autor", () => { const simulator = createSimulator(); // user1 posta, tira, e posta de novo let ledger = simulator.as("user1").postNote("Nota 1"); const commitment1 = ledger.authorCommitment; simulator.as("user1").takeDown(); ledger = simulator.as("user1").postNote("Nota 2"); expect(commitment1).toEqual(ledger.authorCommitment); // agora o user2 posta simulator.as("user1").takeDown(); ledger = simulator.as("user2").postNote("Nota 3"); expect(commitment1).not.toEqual(ledger.authorCommitment); });
1o simulador roda o contrato localmente, sem deploy
2.as("user1") e .as("user2") trocam a chave privada de cada um
3mesmo autor, mesmo commitment; outro autor, commitment diferente

npm test, e a privacidade do slide anterior vira um teste que passa.
